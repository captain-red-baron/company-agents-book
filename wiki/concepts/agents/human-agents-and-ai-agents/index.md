# Human Agents and AI Agents — Anchor Page

The most important insight about AI agents is not technical. It is that we have been building, managing, and delegating to agents for as long as organizations have existed. Every employee is an agent. Every contractor is an agent. Every manager delegating work to a team is a principal creating agents. The entire field of organizational management is, at its core, a set of practices for making agency work at scale.

AI agents are new implementations of an old pattern. The principal-agent problem — that the person doing the work may not perfectly align with the person who assigned it — is the same whether the agent is a human sales rep or an LLM-powered assistant. The mechanisms differ, but the structural challenge is identical.

This is not just an analogy. It is a practical operating insight: the management frameworks companies already use for human agents are directly transferable to AI agents, with adjustments.

---

## The Core Parallel

| Dimension | Human Agent | AI Agent |
|-----------|-------------|----------|
| **Acts on behalf of** | Employer, client, principal | User, system, organization |
| **Perceives through** | Senses, communication, documents | Context windows, APIs, tool outputs |
| **Reasons with** | Experience, judgment, training | LLM inference, prompts, memory |
| **Acts through** | Conversations, decisions, physical actions | Tool calls, API requests, code execution |
| **Bounded by** | Employment contract, fiduciary duty, social norms | Guardrails, system prompts, permission scopes |
| **Evaluated by** | Performance reviews, KPIs, outcomes | Metrics, evaluations, success criteria |
| **Alignment mechanism** | Incentives, culture, supervision | RLHF, constitutional AI, monitoring |
| **Fails when** | Misaligned incentives, poor training, lack of context | Hallucination, goal drift, insufficient guardrails |
